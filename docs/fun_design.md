## 2. 코어 루프 (Core Loop)

```
┌─────────────────────────────────────────────────────────┐
│  1. 시간대/지역 탐험                                     │
│         ↓                                               │
│  2. 자원/정보 수집                                       │
│         ↓                                               │
│  3. 시간 제한/조건 도달 → 루프 종료                      │
│         ↓                                               │
│  4. 메타 성장/정보 유지 → 새 루프 시작 ────→ (1로 반복)   │
└─────────────────────────────────────────────────────────┘
```

### 루프 상세 설명

1. **탐험**: 다양한 시간대/지역 탐험 및 이벤트 경험
2. **자원/정보 수집**: 탐험 중 자원 획득, 중요 정보 발견
3. **루프 종료**: 시간 제한 또는 특정 조건에서 루프 종료
4. **메타 성장**: 영구 강화, 정보 유지 후 새 루프 시작

## 3. 재미 축 (Fun Pillars)

### 3.1 탐험/발견 (Exploration)
- **목표**: 미지의 영역을 탐험하는 재미
- **핵심 경험**: 이런 곳이 있었어? 하는 발견의 기쁨
- **주요 요소**:
  - 다양한 시간대 (고대/현대/미래)
  - 숨겨진 지역
  - 비밀 이벤트
  - 탐험 도감

### 3.2 루프 성장 (Loop Growth)
- **목표**: 루프를 거듭할수록 효율적인 탐험
- **핵심 경험**: 이제 이 루트가 최적이야! 하는 숙달
- **주요 요소**:
  - 메타 업그레이드
  - 지식 유지
  - 단축 경로 해금
  - 영구 능력

### 3.3 빌드/경로 선택 (Route Choice)
- **목표**: 매 루프마다 다른 경로를 선택하는 전략적 재미
- **핵심 경험**: 이번엔 이 루트로 가볼까?
- **주요 요소**:
  - 분기 시스템
  - 시간 배분
  - 자원 트레이드오프
  - 엔딩 분기

## 4. 디자인 레버 (Design Levers)

### 4.1 탐험 구조
| 레버 | 설명 |
|------|------|
| 루트/시간/보상 | 탐험 루트/시간 제한/루프 보상 구조 명확화 |
| 구현 방향 | 루프당 24시간(게임 내), 3개 주요 루트, 루트별 특화 보상 |

### 4.2 회귀 후 차별화
| 레버 | 설명 |
|------|------|
| 시작 조건 변화 | 회귀 후 시작 지점/능력/정보 차별화 |
| 구현 방향 | 10회 루프 → 새 시작점 해금, 특정 발견 → 초기 능력 추가 |

### 4.3 타임라인 UI
| 레버 | 설명 |
|------|------|
| 선택/역사 시각화 | "타임라인" UI로 플레이어의 선택/역사를 시각화 |
| 구현 방향 | 분기점 표시, 이전 루프 기록, 미발견 루트 힌트 |

## 5. 재미 측정 지표 (KPI)

| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| 루프당 탐험 시간 | 20~30분 | 루프 시작 → 종료 |
| 일일 루프 횟수 | 3회 이상 | 루프 시작 로그 |
| 발견 진행률 | 주 10% 이상 | 탐험 도감 달성률 |
| 루트 다양성 | 3루트 이상 사용 | 루트 선택 로그 |
| D7 리텐션 | 30% 이상 | 7일차 잔존율 |
